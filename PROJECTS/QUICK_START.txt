INSTALLATION
============
1. Navigate to project root:
   cd C:\Projects\Crawler\PROJECTS

2. Install test dependencies:
   pip install -r requirements-test.txt

3. Verify installation:
   pytest --version


RUNNING TESTS
=============

OPTION 1: Using Windows Batch Script (Recommended)
   RUN_TESTS.bat                  # Run all tests
   RUN_TESTS.bat scraper          # Run scraper-full tests only
   RUN_TESTS.bat classify         # Run classify tests only
   RUN_TESTS.bat crossref         # Run cross-reference tests only
   RUN_TESTS.bat coverage         # Run with coverage report
   RUN_TESTS.bat help             # Show help

OPTION 2: Using pytest directly
   pytest                         # Run all tests
   pytest -v                      # Verbose output
   pytest --cov=src/services      # With coverage

OPTION 3: By service
   pytest src/services/scraper-full/tests/unit/ -v
   pytest src/services/classify/tests/unit/ -v
   pytest src/services/cross-reference/tests/unit/ -v

OPTION 4: Specific test file
   pytest src/services/scraper-full/tests/unit/test_scraper_validation.py -v

OPTION 5: Specific test class
   pytest src/services/scraper-full/tests/unit/test_scraper_validation.py::TestValidateUrl -v

OPTION 6: Specific test function
   pytest src/services/scraper-full/tests/unit/test_scraper_validation.py::TestValidateUrl::test_valid_https_url -v

TROUBLESHOOTING
===============
Q: pytest command not found
A: Make sure requirements-test.txt is installed: pip install -r requirements-test.txt

Q: Import errors when running tests
A: Verify conftest.py files are in correct locations and __init__.py files exist

Q: Tests fail with "fixture not found"
A: Check fixture name spelling and that conftest.py is in correct directory

Q: Windows batch file not executing
A: Try running with: python -m pytest instead of RUN_TESTS.bat
